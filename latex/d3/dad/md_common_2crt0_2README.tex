 This directory contains some limited C runtime features. It will take care of booting the binary, setting up some minimal environment and then call the main function. It also contains some basic implementations of standard C functions which are required by the compiler when doing self-\/hosted compilation.\hypertarget{md_common_2crt0_2README_autotoc_md1}{}\doxysection{\texorpdfstring{Using very bare C runtime}{Using very bare C runtime}}\label{md_common_2crt0_2README_autotoc_md1}
The file {\ttfamily crt0.\+s} is the entry point of the C runtime. It will set up the stack, and call the {\ttfamily main} function. It will not perform any other initialization beyond clearing the BSS section.\hypertarget{md_common_2crt0_2README_autotoc_md2}{}\doxysection{\texorpdfstring{Using C++ runtime}{Using C++ runtime}}\label{md_common_2crt0_2README_autotoc_md2}
The file {\ttfamily crt0cxx.\+s} is the entry point of the C++ runtime. It will set up the stack, and then go to the {\ttfamily \doxylink{cxxglue_8c}{cxxglue.\+c}} file for further initialization, which will then call the {\ttfamily main} function. The {\ttfamily \doxylink{cxxglue_8c}{cxxglue.\+c}} file also contains the minimal implementation of the C++ runtime that is required by the C++ compiler.\hypertarget{md_common_2crt0_2README_autotoc_md3}{}\doxysection{\texorpdfstring{Memory C runtime functions}{Memory C runtime functions}}\label{md_common_2crt0_2README_autotoc_md3}
The files {\ttfamily \doxylink{memory-c_8c}{memory-\/c.\+c}} and {\ttfamily memory-\/s.\+s} contain the 4 basic memory functions that are required by the self-\/hosted C and C++ compilers. These are {\ttfamily memcpy}, {\ttfamily memset}, {\ttfamily memcmp}, and {\ttfamily memmove}. These are implemented in a very basic way, being very small, and are not optimized for speed. They also contain optimized versions of {\ttfamily memcpy} and {\ttfamily memset} for the MIPS architecture, available using the LDFLAGS {\ttfamily -\/Wl,-\/wrap,memcpy} and {\ttfamily -\/Wl,-\/wrap,memset} toggles. They are somewhat bigger, but much faster. Note that these 4 functions are available directly when writing C or C++ code using the {\ttfamily \+\_\+\+\_\+builtin\+\_\+memcpy}, {\ttfamily \+\_\+\+\_\+builtin\+\_\+memset}, {\ttfamily \+\_\+\+\_\+builtin\+\_\+memcmp} and {\ttfamily \+\_\+\+\_\+builtin\+\_\+memmove} functions, with no need to include any header file.
