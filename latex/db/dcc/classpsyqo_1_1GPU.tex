\doxysubsection{Detailed Description}
The singleton \doxylink{classpsyqo_1_1GPU}{GPU} class. 

This class shouldn\textquotesingle{}t be instantiated directly. It is a singleton instantiated within the {\ttfamily \doxylink{classpsyqo_1_1Application}{Application}} class, and accessed using the {\ttfamily gpu} method. It contains the current state of the psyqo renderer, and provides various helpers for rendering.

Single-\/use timers will automatically be disabled upon being fired, and their id will no longer be valid. The returned id is guaranteed to be unique across active timers, but may collision with the id of timers that got canceled or got disabled on their own.

Gets the current timestamp in microseconds. 

The current timestamp is in microseconds. It will wrap around after a bit more than an hour, so it shouldn\textquotesingle{}t be used for deadlines that are more than 30 minutes away. This relies on root counter 1 set in hsync mode without any target value. The value will be updated during the idle moments of the page flip, without relying on interrupts. The precision isn\textquotesingle{}t really good, as it assumes one scanline runs at 64us, but it should be good enough for most purposes. Creating a stopwatch out of it should show that it\textquotesingle{}s running a bit too fast, approximately 1 second too fast every minute or so. Its monotonicity should be proper however.

The method {\ttfamily \doxylink{classpsyqo_1_1GPU_a8f9c350c46e0b138867bb2289c407012}{now()}} should be reserved for interacting with timers. If longer span is required, with more accuracy but less precision, for something that\textquotesingle{}s not related with timers, then the current amount of time in seconds since the application started can simply be obtained using {\ttfamily \doxylink{classpsyqo_1_1GPU_a43245681eb3f72817ab68f0c24e2d04b}{get\+Frame\+Count()} / \doxylink{classpsyqo_1_1GPU_aa6cecf7edc1d353b0571063b92e9b95b}{get\+Refresh\+Rate()}}. \begin{DoxyReturn}{Returns}

Uploads a buffer to the VRAM as a non-\/blocking call. 

This method will initiate an upload of the specified set of pixels to the VRAM, at the specified location and size. The \doxylink{classpsyqo_1_1GPU}{GPU} cache will be flushed. It will return immediately, and the upload will be performed in the background. Upon completion, the specified callback will be called. If {\ttfamily dma\+Callback} is set to {\ttfamily FROM\+\_\+\+ISR}, the callback will be called from the interrupt handler, and care must be taken to properly synchronize variable changes. Please use the EASTL\textquotesingle{}s {\ttfamily atomic\+\_\+signal\+\_\+fence} function for this purpose. If {\ttfamily dma\+Callback} is set to {\ttfamily FROM\+\_\+\+MAIN\+\_\+\+LOOP}, the callback will be called in the same execution context as the main loop, and it is therefore safe to access variables there. The callback will thus be called between calls to the current scene\textquotesingle{}s {\ttfamily frame} method, or during {\ttfamily Kernel\+::pump\+Callbacks()}. Note that during the upload, no \doxylink{classpsyqo_1_1GPU}{GPU} operation should be performed.
