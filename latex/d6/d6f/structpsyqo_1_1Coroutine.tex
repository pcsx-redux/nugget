\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename T = void$>$\newline
struct psyqo\+::\+Coroutine$<$ T $>$}
A suitable type to hold and return a C++20 coroutine. 

C++20 introduced the concept of coroutines in the language. This type can be used to properly hold a coroutine and yield and resume it within psyqo. An important caveat of using coroutines is that the language insist on calling {\ttfamily new} and {\ttfamily delete} silently within the coroutine object. This may be a problem for users who don\textquotesingle{}t want to use the heap.

Returns the value returned by the coroutine. 

This method returns the value returned by the coroutine. It is only valid to call it after the coroutine has finished executing. The typical usage of this method is to call it after the {\ttfamily done} method returns {\ttfamily true}. The coroutine sets its return value using the {\ttfamily co\+\_\+return} keyword. Since it is possible for the return type to be {\ttfamily void}, the return type of this method is {\ttfamily T} if {\ttfamily T} is not {\ttfamily void}, and {\ttfamily \doxylink{structpsyqo_1_1Coroutine_1_1Empty}{Empty}} if {\ttfamily T} is {\ttfamily void}.
