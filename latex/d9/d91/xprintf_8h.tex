Prints a formatted string to a callback. 

This function behaves mostly as you\textquotesingle{}d expect from a typical printf, with some extra formatting options\+: The \textquotesingle{}=\textquotesingle{} flag (similar to \textquotesingle{}-\/\textquotesingle{}) causes the output to be centered in the appropriately sized field. The b field outputs an integer in binary notation. The c field now accepts a precision. The character output is repeated by the number of times the precision specifies. The \textquotesingle{} field works like c, but takes as its character the next character of the format string, instead of the next. This is useful for when using the precision modifier. For example, `printf("{}\%.\+78'-\/"{})\`{} prints 78 minus signs, the same as \`{}printf("{}\%.\+78c"{}, \textquotesingle{}-\/\textquotesingle{}){\ttfamily would. S will display a string that has its control characters escaped using the caret notation. For example, character 26, also known as EOF, will be displayed as \texorpdfstring{$^\wedge$}{\string^}Z. z will display a string, and immediately dispose of it using }\doxylink{psyqo_2alloc_8h_a2e8fd619ab9a5b9581d93558dfb4ee76}{psyqo\+\_\+free()}{\ttfamily . r will display a number as an English ordinal. For example, 1 will be displayed as "{}1st"{}, 2 as "{}2nd"{}, etc... Finally, one difference with normal printf, is that \%\#x will output "{}0x0"{} instead of "{}0"{} for the value 0. Note that floating point formatting will not be available. The callback function will be called with the string to print, and the length of the string. The third argument will be the opaque pointer passed through. The}xprintf\`{} variant is available.
