18 // Description
19 //
20 // EASTL includes a fairly serious type traits library that is on par with the
21 // one found in Boost but offers some additional performance-enhancing help as well.
22 // The type_traits library provides information about class types, as opposed to
23 // class instances. For example, the is_integral type trait tells if a type is
24 // one of int, short, long, char, uint64_t, etc.
25 //
26 // There are three primary uses of type traits:
27 // * Allowing for optimized operations on some data types.
28 // * Allowing for different logic pathways based on data types.
29 // * Allowing for compile-type assertions about data type expectations.
30 //
31 // Most of the type traits are automatically detected and implemented by the compiler.
32 // However, EASTL allows for the user to explicitly give the compiler hints about
33 // type traits that the compiler cannot know, via the EASTL_DECLARE declarations.
34 // If the user has a class that is relocatable (i.e. can safely use memcpy to copy values),
35 // the user can use the EASTL_DECLARE_TRIVIAL_RELOCATE declaration to tell the compiler
36 // that the class can be copied via memcpy. This will automatically significantly speed
37 // up some containers and algorithms that use that class.
38 //
39 // Here is an example of using type traits to tell if a value is a floating point
40 // value or not:
41 //
42 // template <typename T>
43 // DoSomething(T t) {
44 // assert(is_floating_point<T>::value);
45 // }
46 //
47 // Here is an example of declaring a class as relocatable and using it in a vector.
48 //
49 // EASTL_DECLARE_TRIVIAL_RELOCATE(Widget); // Usually you put this at the Widget class declaration.
50 // vector<Widget> wVector;
51 // wVector.erase(wVector.begin()); // This operation will be optimized via using memcpy.
52 //
53 // The following is a full list of the currently recognized type traits. Most of these
54 // are implemented as of this writing, but if there is one that is missing, feel free
55 // to contact the maintainer of this library and request that it be completed.
56 // As of this writing all C++11 type traits are supported, as well as some additional ones.
57 // http://en.cppreference.com/w/cpp/types
58 //

196 // the user can know if it is such. Thus EASTL_DECLARE_TRIVIAL_RELOCATE is provided to

206 // Requirements
207 //
208 // As of this writing (5/2005), type_traits here requires a well-conforming
209 // C++ compiler with respect to template metaprogramming. To use this library
210 // you need to have at least one of the following:
211 // MSVC++ 7.1 (includes Win32, Win64, and WinCE platforms)
212 // GCC 3.2 (includes MacOSX, and Linux platforms)
213 // Metrowerks 8.0 (incluees MacOSX, Windows, and other platforms)
214 // EDG (includes any compiler with EDG as a back-end, such as the Intel compiler)
215 // Comeau (this is a C++ to C generator)
216 //
217 // It may be useful to list the compilers/platforms the current version of
218 // type_traits doesn't support:
219 // Borland C++ (it simply has too many bugs with respect to templates).
220 // GCC 2.96 We used to have a separate set of type traits for this compiler, but removed it due to lack of necessity.
222 
224 // Implementation
225 //
226 // The implementation here is almost entirely based on template metaprogramming.
227 // This is whereby you use the compiler's template functionality to define types
228 // and values and make compilation decisions based on template declarations.
229 // Many of the algorithms here are similar to those found in books such as
230 // "Modern C++ Design" and C++ libraries such as Boost. The implementations here
231 // are simpler and more straightforward than those found in some libraries, due
232 // largely to our assumption that the compiler is good at doing template programming.
